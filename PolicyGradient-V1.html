Policy Gradient Method - REINFORCE

This notes make more sense if you have read previous notes or have a understanding of Machine learning and Reinforcement learning understanding.

DQN is a value-based reinforcement learning algorithms, where optimal policy is computed by iteratively updating optimal QQQ-function(QQQ-values) . i.e, we compute the QQQ-values of all state-action pairs to find the policy and we select an action in each state which has maximum QQQ-value.

Disadvantages of the value-based method like DQN is it works well for environments with a discrete action space, like grid world. But struggles when environments requires continuous action space or have a high dimension, like robot learning to walk.

Where as Policy Gradient methods compute the optimal policy without having to compute the optimal QQQ-function. Policy-based methods works well for both discrete and continuous action spaces. Policy-based methods usually employs stochastic policy so it does good in exploration- exploitation trade-off implicitly, but deterministic policy can also be used.

Policy gradient

In Policy gradient method we try to find the optimal parameter θ\thetaθ of the neural network, as to get correct probability distribution over the action space for a given state. So it assigns high probabilities to actions that maximise the expected return of the trajectory, to achieve this we maximise the objective function by performing gradient ascent.

We can summarise the steps involved in Vanilla Policy Gradient

  • Initialise the policy network parameter θ\thetaθ.

  • By following policy πθ\pi_{\theta}πθ​ we generate N number of trajectories (τi)i=1N(\tau^i)_{i=1}^N(τi)i=1N​.

  • Compute the return of the given trajectory R(τ)R(\tau)R(τ), which is reward for the step. Can be positive , negative or neutral.

  • Calculate the gradients ∇θJ(θ)=1N∑i=1N∑t=0T−1∇θlogπθ(at∣stR(τ))\nabla_{\theta} J(\theta) = \frac{1}{N} \sum_{i=1}^N \sum_{t=0}^{T-1} \nabla_{\theta} log\pi_{\theta}(a_{t}|s_{t} R(\tau))∇θ​J(θ)=N1​i=1∑N​t=0∑T−1​∇θ​logπθ​(at​∣st​R(τ))
    logπθ(at∣st)log\pi_{\theta}(a_{t}|s_{t})logπθ​(at​∣st​) is log probability of taking an action aaa given the state sss at a time ttt, R(τ)R(\tau)R(τ) is the return of the trajectory/episode. If the episode outputs a negative return, i.e R(τ)=−1R(\tau) = -1R(τ)=−1, then probability of all the actions that we took in each state of that episode is decreased. Similarly if R(τ)R(\tau)R(τ) is positive then probability of all the actions that we took in each state of that episode is increased. Expectation is averaged using Monte Carlo approximation, i.e sum over N trajectories.

  • Performing gradient ascent network parameters are updated by below equation, for each state in the episode/trajectory, equation for the same.
    θ=θ+α∇θlogπθ(at∣st)R(τ)\theta = \theta + \alpha \nabla_{\theta} log \pi_{\theta}(a_{t} | s_{t}) R(\tau)θ=θ+α∇θ​logπθ​(at​∣st​)R(τ) which can be written also as θ=θ+α∇θJ(θ)\theta = \theta + \alpha\nabla_{\theta} J(\theta)θ=θ+α∇θ​J(θ)

Update the gradients in such a way that actions yielding a high return will get a high probability, and actions yielding a low return will get a low probability.

REINFORCE with baseline

In previous section we saw that same policy with which we are generating episodes in every iteration is improved. So our return varies greatly in each episode thus introducing high variance in the gradient updates, Which leads to lot of time for convergence. To reduce variance we can use reward to go method where reward for each step is used instead of reward of trajectory.
∇θj(θ)=∇θlogπθ(a0∣s0)R0+∇θlogπθ(a1∣s1)R1+...+∇θlogπθ(aN∣sN)RN\nabla_{\theta} j(\theta) =\nabla_{\theta} log \pi_{\theta}(a_{0} | s_{0}) R_{0} + \nabla_{\theta} log \pi_{\theta}(a_{1} | s_{1}) R_{1} + ...+ \nabla_{\theta} log \pi_{\theta}(a_{N} | s_{N}) R_{N}∇θ​j(θ)=∇θ​logπθ​(a0​∣s0​)R0​+∇θ​logπθ​(a1​∣s1​)R1​+...+∇θ​logπθ​(aN​∣sN​)RN​

earlier we used R(τ)R(\tau)R(τ) every where. Reward to go is defined as
Rt=∑t′=0T−1r(st′,at′)R_{t} = \sum_{t' = 0}^{T - 1} r(s_{t}^{'}, a_{t}^{'})Rt​=t′=0∑T−1​r(st′​,at′​)

Thus we can write θ=θ+α∇θlogπθ(at∣st)Rt\theta = \theta + \alpha \nabla_{\theta} log \pi_{\theta}(a_{t} | s_{t}) R_{t}θ=θ+α∇θ​logπθ​(at​∣st​)Rt​

To help deal with variance further we now introduce a new function called a baseline function and we subtract baseline bbb from the reward-to-go RtR_{t}Rt​,
∇θj(θ)=1N∑i=1N∑t=0T−1∇θlogπθ(at∣st)(Rt−b)\nabla_{\theta} j(\theta) = \frac{1}{N} \sum_{i=1}^N \sum_{t=0}^{T-1} \nabla_{\theta} log\pi_{\theta}(a_{t}|s_{t}) (R_{t} - b)∇θ​j(θ)=N1​i=1∑N​t=0∑T−1​∇θ​logπθ​(at​∣st​)(Rt​−b)

Simple base line can be b=1N∑i=1NR(τ)b = \frac{1}{N}\sum_{i=1}^{N} R(\tau)b=N1​∑i=1N​R(τ). But we should take care while introducing baseline function, that it should not depend on network parameters. Other popular baseline function is value function, advantage function, QQQ-function, so on.

We go with value function as our baseline function, another network to learn the value of the state, which is parameterised by ϕ\phiϕ, which is trained using MSEMSEMSE loss function between actual return RtR_{t}Rt​ and predicted value return Vϕ(st)V_{\phi}(s_{t})Vϕ​(st​).

Now the objective function of the value network can be defined as,

J(ϕ)=1N∑i=1N∑t=0T−1(Rt−Vϕ(St))2J(\phi) = \frac{1}{N} \sum_{i=1}^N \sum_{t=0}^{T-1} (R_{t} - V_{\phi}(S_{t}))^2J(ϕ)=N1​i=1∑N​t=0∑T−1​(Rt​−Vϕ​(St​))2

using gradient descent we can minimise the error and update the network parameter as given below

ϕ=ϕ−α∇ϕJ(ϕ)\phi = \phi - \alpha\nabla_{\phi} J(\phi)ϕ=ϕ−α∇ϕ​J(ϕ)

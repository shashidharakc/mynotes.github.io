Now lets check how we can train our Deep learning network using all the information we have to estimate QQQ-value for a given state and action pair

Replay Buffer

To Train our network more effectively we collect agents experience, i.e transition information (s,a,r,s′)(s, a, r, s')(s,a,r,s′) in a queue structure (first in first out, FIFO), called Replay buffer denoted by DDD. When we have accumulated sufficient transitions we sample a random mini batch from the queue and train our network. This buffer will store only a predefined number of the agent’s experience, as a result when the buffer is full we replace the old experience with new experience.

Our goal is to predict the QQQ-value, which is a continuous value, a regression task using Deep neural network. Mean squared error (MSE) suits well as the loss function for the regression task. Equation for MSE is as given below,

Nothing but sum of average squared difference between the target value, yyy and predicted value, y^\hat yy^​. NNN in the number of samples.

Target network

As the predicted and target values both depend on the same parameter θ\thetaθ . If we update θ\thetaθ, then both the target and predicted values change. Thus, the predicted value keeps on trying to be the same as the target value, but the target value keeps on changing due to the update on the network parameter θ\thetaθ.

To solve this issue we will introducing another neural network called a target network, which is a copy of main network, for computing the QQQ-value of the next state-action pair in the target. By freeze the target value for predefined number of steps and compute only the predicted value so that our predicted value matches the target value. The parameter of the target network is represented by θ′\theta'θ′. The target network is frozen for a while and then the target network parameter θ′\theta 'θ′ is updated by just copying the main deep Q network parameter θ\thetaθ.

first we initialise the Q function with random values or with zeros. Then we extract a policy from this randomly initialised Q function and act in the environment.Our initial policy will definitely not be optimal as it is extracted from the randomly initialised Q function, but on every episode, we will update the Q function (Q values). The objective of reinforcement learning is to find the optimal policy, that is, the
policy that gives us the maximum return (the sum of rewards of the episode). In
order to compute the policy, first we compute the Q function. Once we have the Q
function, then we extract the policy by selecting an action in each state that has the
maximum Q value.

In DQN, we used a neural network to approximate the Q function (Q value).
Given a state, the network will return the Q values of all possible actions in that
state. For instance, consider the grid world environment. Given a state, our DQN will
return the Q values of all possible actions in that state. it is difficult to compute the Q function when our environment has a large number of states and actions as it would be expensive to compute the Q values of all possible state-action pairs. So, we resorted to the Deep Q Network (DQN)

The objective of reinforcement learning is to find maximum return (the sum of rewards of the episode), using the optimal policy.
